# Frontend Project Guidelines

## Project Stack
- **Framework**: React 18+ with TypeScript
- **Build Tool**: Vite
- **UI Library**: Chakra UI
- **Package Manager**: npm
- **Linter**: ESLint

## Code Style & Conventions

### Component Structure
- Use functional components with hooks
- Place components in `/src/components/` organized by feature or type (ui/, app/)
- Each component should have a single responsibility
- Use TypeScript for type safety

### File Organization
```
src/
├── components/          # React components
│   ├── ui/             # Reusable UI components
│   └── app/            # Feature/page components
├── pages/              # Page components (if used)
├── hooks/              # Custom React hooks
├── services/           # API services
├── types/              # TypeScript type definitions
├── utils/              # Utility functions
├── styles/             # Global styles if needed
└── App.tsx             # Root component
```

### Component Naming
- Use PascalCase for component names
- Component files should match component name
- Prefer `index.tsx` for component directories

### TypeScript Best Practices
- Always define prop types with interfaces/types
- Avoid using `any` type
- Use strict mode in tsconfig.json
- Define return types for functions

### Chakra UI Integration
- Import components from `@chakra-ui/react`
- Use Chakra's theming system for consistency
- Leverage Box, Flex, Grid for layout
- Use Chakra's color tokens for styling
- Prefer props over sx prop when possible

### CSS & Styling
- Prefer Chakra UI components and props over CSS classes
- Keep custom CSS minimal and in separate files if needed
- Use Chakra's responsive props (e.g., `display={{ base: 'block', md: 'flex' }}`)
- Follow color scheme: use theme colors consistently

## Development Workflow

### Running the Project
do not use any npm or docker command

### Git Practices
- Write clear, descriptive commit messages
- Keep commits focused and atomic
- Use feature branches for new features

## Performance Considerations
- Code split large components using React.lazy()
- Optimize images and assets
- Use React DevTools Profiler to identify bottlenecks
- Minimize bundle size

## Testing (If Applicable)
- Place test files next to components with `.test.tsx` extension
- Use consistent testing patterns

## Common Fixes

### Import Issues
- Always import React components properly
- Import Chakra components from `@chakra-ui/react`

### Type Safety
- Define component prop interfaces
- Use optional (?) and required properties correctly

### Chakra UI Props
- Use responsive arrays for responsive design: `width={['100%', '50%', '30%']}`
- Use shorthand Chakra props: `mt` (marginTop), `p` (padding), etc.

## API Integration
- Create service files in `/src/services/` for API calls
- Use async/await patterns
- Handle errors appropriately
- Consider using a request client (fetch or axios wrapper)

## Accessibility
- Use semantic HTML elements
- Include alt text for images
- Ensure proper heading hierarchy
- Test keyboard navigation
- Use Chakra's built-in accessibility features

## Form Handling
- Use Chakra UI Form components (FormControl, FormLabel, etc.)
- Implement proper validation
- Provide clear error messages to users

## Notes
- This is a Vite project, so HMR (Hot Module Replacement) is enabled by default
- Keep components small and reusable
- Collaborate with backend team on API contracts (types/interfaces)
